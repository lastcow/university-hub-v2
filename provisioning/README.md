# `provisioning/`

Generated by [`scripts/provision-university.mjs`](../scripts/provision-university.mjs).

Each customer slug gets its own subdirectory with a `wrangler.toml` that
the deploy / secret / migration commands target via `--config=`:

```text
provisioning/
  acme/
    wrangler.toml
  beta-state/
    wrangler.toml
```

Subdirectories are gitignored (see [`.gitignore`](../.gitignore)). The
script regenerates them deterministically from the same inputs, so a new
operator can recreate any tenant's config with:

```bash
node scripts/provision-university.mjs \
  --name="…" --slug=<slug> \
  --admin-email=… --admin-name=… \
  --skip-bootstrap --skip-pages-deploy --dry-run
```

The dry-run path writes the toml without touching live Cloudflare
resources.

See [`docs/per-customer-provisioning.md`](../docs/per-customer-provisioning.md)
for the full walkthrough.
